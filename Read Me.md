# QA Portfolio

This repository showcases my manual testing skills through structured test cases, execution results, and defect reporting.

## 📌 Project Overview
This project focuses on testing the registration form functionality of a web application using manual testing techniques. The goal was to validate correct behavior, identify input validation issues, and document results clearly.

## 🧪 Features Tested
- Registration form functionality  
- Valid form submission scenario  
- Invalid input scenarios (email, phone number)  
- Required field validation  

## 📂 Repository Structure
- `test-cases/` → Contains manual test cases for registration functionality  
- `bug-reports/` → Contains documented defects and observations  
- `screenshots/` → Contains screenshots as evidence of test execution  

## 🛠️ Skills Demonstrated
- Test case design  
- Test execution  
- Positive and negative testing  
- Validation testing  
- Bug reporting  
- Clear documentation  

## 🧠 Testing Techniques Used
- Equivalence Partitioning  
- Boundary Value Analysis  
- Error Guessing  

## 📅 Test Execution Details
- Application tested: demoqa.com (demo web application)  
- Tests executed manually  
- Results recorded with pass/fail status and actual outcomes  

## 📸 Evidence
Screenshots are included in the `screenshots/` folder to support test execution results and identified defects.
